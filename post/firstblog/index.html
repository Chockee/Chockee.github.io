如何用Hugo+GitPage搭建静态博客

Written by ZhuJ1ntao with ♥ on November 25, 2019 in 练级之路
  • 作为大三党一名，再三考虑后选择放弃考研，加入明年的秋招大军。
  • 准备用博客记录下这一年中的学习经历，顺便培养自己的总结、表达能力。 于是上网搜寻了个人博客的搭建方法，主要的方法是用Hexo或者Hugo在本地搭建静态页面，然后再部署到服务器上。而Hugo较于Hexo更加灵活，快速，于是选择了Hugo。 在搭建的过程中也遇到了不少坑，主要是自己对hugo和git命令不熟悉的原因，百度也很少方案，最后靠自己慢慢摸索解决后，才发现百度上找不到的原因是这些问题实在太简单解决了…

下载Hugo

  • 我用的是Windows系统，在Github上的Hugo中，找到跟自己系统相对应的Hugo版本下载即可，我的系统是Windows64位，所以下载windows-64bits的版本。
  • 下载以后解压即可。注意，直接打开Hugo.exe是没用的！！要在系统的环境变量的PATH中添加Hugo.exe所在文件夹的路径。 比如Hugo.exe的路径是D:/Hugo/Hugo.exe，那么我们只需要填写D:/Hugo/就可以了
  • 然后打开cmd，输入hugo version查看是否安装成功即可。

新建自己的博客目录及设置主题

  • 在命令行界面中，进入到你想保存博客的目录。输入
    hugo new site [foldername]
    创建博客根目录文件夹

  • 可以在https://themes.gohugo.io选择自己喜欢的主题，点击进入介绍页面，下拉可以看到类似于

    cd themes
git clone <~~~~>

    的段落，我们打开cmd，跟着操作即可。（Tips:如果博客页面根目录下没有themes文件夹，请自行创建）

  • 在命令行中输入
    hugo server -t [theme_name] --buildDrafts
    可以在本地搭建博客页面，然后在浏览器的地址栏中输入http://localhost:1313/，可以预览页面，Ctrl+C可以退出预览

利用GitPage部署博客

  • 首先要有一个github账号
    (Windows系统还需要安装Git工具)。
  • 登陆github，新建一个Repository 注意Repository name的填写内容，一定要填写自己的[用户名]+github.io，然后创建。
  • 打开cmd，cd进入到博客根目录下，输入
    hugo --theme=[theme_name] --baseUrl="https://[username].github.io" --buildDrafts
    可以看到一个public文件夹就被创建了。

  • 在命令行界面中，依次输入：

    cd public  
git init   #初始化一个新的本地仓库
git add .   #将当前文件夹的所有文件都添加到暂存区
git commit -m "first commit"    #将暂存区的文件提交到数据目录
git remote add origin https://github.com/[Username]/[Username].github.io.git"  #连接到我的仓库
git push -u master   #把文件推送到我仓库的master目录下，同时关联本地和远程的master分支

    至此，博客部署完成，在地址栏中输入
    https://[username].github.io 即可访问。

关于后续的主题更新以及博客更新

  • 在https://themes.gohugo.io选择喜欢的主题，然后下载到本地的themes文件夹中

  • 那么如何把主题改成自己喜欢的样式呢？这个时候Hugo的好处就体现出来了，我们先将主题相关的配置文件拷贝到我们博客的根目录下，然后打开cmd，输入并运行

    hugo server

    然后用浏览器进入http://localhost:1313/，这个时候我们可以根据主题的说明文档对博客进行修改啦(小白推荐从config.toml文件入手，更改baseUrl、头像、联系方式等)，并且可以实时查看我们的修改结果。将博客改成我们心仪的样式之后，Ctrl+C结束预览。 有的主题可能会报错，会提示hugo的版本不支持当前主题之类的，这个时候需要下载Hugo的拓展工具，我在折腾了一阵后放弃了，简单粗暴的选择了其他适合当前Hugo版本号的主题。

  • 更新好主题后我们怎么就给我们的博客写第一篇文章吧（就是我这篇了），在博客的根目录下，在命令行中输入

    hugo new post/[name].md

    然后我们就可以在./content/post中找到我们的markdown格式的文件啦，我们可以简答粗暴的用IDE或者下载Notepad++打开它，然后撰写完成自己的第一篇博客。
    (Windows系统不推荐用记事本编辑，因为它会在每个文件的开头添加0xefbbbf字符，从而带来一些难以捉摸的问题)

  • 主题、文章都有了，我们就该尝试把它们更新到Github上。 在命令行中输入以下内容就可以完成更新：

    hugo 
cd public
git add .
git commit -m "(*&$@*(&!"
git push

以上就是基于Hugo的个人博客搭建以及更新的简单教程啦。
